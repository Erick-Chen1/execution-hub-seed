# 实施计划（10 天）

1. 第 1 天
   - Workflow/Task/Step/Executor 数据模型与 API 草案
   - 开发任务切分与责任人分配

2. 第 2 天
   - 数据库迁移脚本
   - Repository 接口与基础 CRUD

3. 第 3-4 天
   - Workflow 定义与版本管理
   - Task 实例化与任务状态推进

4. 第 5-6 天
   - Orchestrator MVP
   - 步骤依赖与并行策略
   - 超时/重试/升级触发

5. 第 7 天
   - Action/Notification 集成
   - 人机交接闭环

6. 第 8 天
   - Evidence 聚合与审计串联
   - 任务级 Evidence API

7. 第 9 天
   - UI 展示与回放
   - Demo 场景适配

8. 第 10 天
   - 全链路联调与演示脚本
   - 文档补全与交付
