# 需求清单

功能需求
- 任务模型: 任务包含多个步骤，支持串行/并行/条件分支。
- 统一执行者: 步骤可指定 AI 或人类执行者。
- 证据链: 每个步骤必须产出证据（AI 输出、日志、文件、人类确认）。
- 失败语义: 支持超时、重试、改派与升级。
- 人机接力: AI -> 人复核 -> AI 继续；或人 -> AI 自动补全。
- 回放审计: 支持按任务/步骤回放执行时间线。

非功能需求
- 可追溯: 全链路 Trace ID。
- 可验证: Evidence Bundle 可校验完整性。
- 可扩展: 新增执行者类型、步骤类型不需改动核心模型。
- 可移植: 核心模型与执行逻辑可直接嵌入工厂 OS。

合规与审计
- 关键操作必须记录审计日志。
- 高风险操作需标记并可查询。
